# LaTeX 通用模板 (电子信息/嵌入式方向)

这是一个专为电子信息、嵌入式开发方向定制的 LaTeX 模板，深度优化了 VS Code + LaTeX Workshop 的使用体验。

## ✨ 核心特性

1.  **极致整洁的工作区**
    *   所有编译产生的中间文件（`.aux`, `.log`, `.toc` 等）全部自动生成在 `out/` 目录中，根目录不再杂乱无章。
    *   **自动隐藏关键文件**：`main.synctex.gz` 会被自动复制回根目录并**设为隐藏属性**。既保证了 VS Code 的“反向跳转”（Ctrl+点击 PDF 跳转代码）功能完美可用，又在视觉上保持了根目录的清爽。

2.  **全自动目录结构镜像**
    *   编译前会自动调用脚本，将源码目录结构“镜像”到 `out/` 目录。
    *   这意味着您可以随意在 `latex/` 下新建子文件夹并使用 `\include`，**无需手动在 out 目录创建对应文件夹**，彻底告别 `I can't write on file` 错误。

3.  **工程化参考文献配置**
    *   预配置 `biblatex` + `biber` 后端。
    *   采用 **IEEE 标准格式**。
    *   **按引用顺序排序 (Sorting=none)**：符合嵌入式/电子类工程论文的行业规范（而非按作者首字母排序）。
    *   智能补全：VS Code 可自动读取 `bib/` 目录下的文献条目。

## 🚀 快速开始

### 1. 环境准备
*   **TeX 发行版**：TeX Live (推荐 2024 或更高版本)
*   **编辑器**：Visual Studio Code
*   **插件**：LaTeX Workshop

### 2. 编译方式
*   打开 `main.tex`。
*   点击左侧 LaTeX 面板的 **Build LaTeX project**。
*   或者直接按 `Ctrl+S` 保存文件（如果开启了自动编译）。
*   *初次编译可能较慢，请耐心等待。*

### 3. 写作指南
*   **主控文件**：`main.tex` —— 负责组织结构，尽量少写正文。
*   **全局配置**：`latex/config.tex` —— 宏包加载、页面设置、自定义命令都在这里。
*   **章节写作**：在 `latex/` 目录下创建新的文件夹和 `.tex` 文件，然后在 `main.tex` 中使用 `\include{latex/chapterX/filename}` 引入。
*   **参考文献**：
    *   将 `.bib` 文件放入 `bib/` 目录（默认为 `bib/ref.bib`）。
    *   在正文中输入 `\cite{key}` 即可引用。
*   **图片资源**：建议存放在 `figures/` 目录。

## 📂 目录结构

```text
.
├── main.tex            # 项目入口文件
├── main.pdf            # (编译后生成) 最终论文
├── bib/                # 参考文献数据库 (.bib)
│   └── ref.bib
├── figures/            # 图片资源存放处
├── latex/              # 源代码目录
│   ├── config.tex      # 导言区配置文件
│   ├── chapter1/       # 第一章
│   ├── chapter2/       # 第二章
│   └── ...
└── out/                # (自动生成) 所有的编译中间文件
```

## ⚠️ 常见问题

*   **Q: 为什么我看不到 `main.synctex.gz` 文件？**
    *   A: 为了美观，它被脚本自动设为“隐藏”了。如果您需要在资源管理器中看到它，请在 Windows 文件夹选项中开启“显示隐藏的文件”。**请勿删除它**，否则无法从 PDF 跳转回代码。

*   **Q: 参考文献显示 `[1]` 但没有内容？**
    *   A: 请检查 `bib/ref.bib` 语法是否正确，或者查看 `out/main.blg` 日志文件寻找 Biber 报错信息。
